COMPATIBILITY LAYER IMPLEMENTATION
===================================

DATE: 2026-06-26
TASK: Task 6 - Implement compatibility layer for Product Discovery response format

OBJECTIVE
---------
Separate the new response format (products + related) for the external Product Discovery API
while maintaining backward compatibility for internal tools like Task Tool.

REQUIREMENTS
------------
1. External Product Discovery API → {"products": [...], "related": [...], "total": ..., "related_total": ...}
2. Task Tool → flat list of RELEVANT products only (previous behavior)
3. No changes to Task Tool's validation logic
4. No changes to retrieval, semantic search, hybrid ranking, or validator

SOLUTION APPROACH
-----------------
Added a `return_format` parameter to `HybridSearchService.search()`:
- return_format='dict' (default) → returns {'relevant': [...], 'related': [...]}
- return_format='flat' → returns flat list of RELEVANT products only

This allows different callers to request the format they need without changing
the underlying pipeline logic.

FILES MODIFIED
--------------

1. backend/services/hybrid_search.py
   - Added `return_format: str = 'dict'` parameter to search() method
   - Added Union[Dict[str, List[Dict]], List[Dict]] return type
   - Added import for Union from typing
   - Added return format handling at the end of search() method:
     * If return_format='flat': return candidates['relevant'] (flat list)
     * If return_format='dict': return candidates dict (default)
   
2. backend/tools/search_tool.py
   - Modified search_service.search() call to explicitly pass return_format='dict'
   - Already had logic to handle dict format and build SearchResponse with products + related
   
3. backend/tools/task_tool.py
   - Modified search_service.search() call to pass return_format='flat'
   - No other changes to Task Tool logic
   - Continues to receive flat list of RELEVANT products as before

## hybrid_search.py

### Method signature:
```python
def search(
    self,
    sport: str,
    category_level_1: Optional[str] = None,
    category_level_2: Optional[str] = None,
    keywords: Optional[List[str]] = None,
    price_limit: Optional[float] = None,
    top_k: int = 10,
    return_format: str = 'dict'
) -> Union[Dict[str, List[Dict]], List[Dict]]:
```

### Return logic:
```python
# Handle different return formats for backward compatibility
if return_format == 'flat':
    # For backward compatibility (Task Tool): return only RELEVANT products as flat list
    if isinstance(candidates, dict):
        relevant = candidates.get('relevant', [])
        logger.info(f"   [COMPAT] Returning flat list: {len(relevant)} RELEVANT products")
        return relevant
    else:
        # Shouldn't happen, but handle gracefully
        return candidates
else:
    # Default: return dict format for external API
    return candidates
```
