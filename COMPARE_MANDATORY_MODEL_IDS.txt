COMPARISON TOOL: MANDATORY MODEL IDENTIFIER MATCHING
====================================================

DATE: 2026-06-26
FIX: Enforce mandatory model identifier matching in comparison resolver

PROBLEM
-------
Query: "Kiprun KD900X LD+"
Incorrect result: "KIPRUN Running Rain Cap"
Reason: Brand matched, model identifier ignored

The resolver was treating model identifiers as optional and allowing
brand similarity to compensate for missing model codes.

SOLUTION
--------
Model identifiers are now MANDATORY matches.

Rule: If query contains model identifiers, candidates MUST contain ALL of them.
      Brand similarity cannot compensate for missing model identifiers.
      Candidates missing any model identifier receive score 0.0 (discarded).

IMPLEMENTATION
--------------

1. Enhanced Model Identifier Extraction
   - Pattern 1: Standard codes (KD900X, MT500, NH100, KS900)
   - Pattern 2: Versions (KD500 2, NH100 3)
   - Combined identifiers when version present (e.g., "KD500 2" becomes single identifier)

2. Mandatory Matching Logic
   - Check: ALL model identifiers must be present in product name
   - Missing ANY identifier → score 0.0 immediately
   - No fallback to brand matching
   - Flexible matching: handles "KD500-2", "KD500 2", "KD500/2" variations

3. Scoring (only if model IDs match)
   - Model IDs present + exact name: 1.0
   - Model IDs present + high overlap: 0.95
   - Model IDs present + good keywords: 0.9
   - Model IDs present + weak keywords: 0.7-0.8
   - No model IDs in query: fuzzy brand+keyword matching (0.3-0.7)

SCORING LOGIC DETAIL
--------------------

Step 1: Check mandatory model identifiers
```python
if model_identifiers:
    for model_id in model_identifiers:
        if model_id NOT in product_name:
            return 0.0  # MANDATORY FAIL
```

Step 2: If all model IDs present, score based on quality
```python
if exact_name_match:
    return 1.0
elif high_overlap (>70%):
    return 0.95
elif good_keyword_overlap (>60%):
    return 0.9
elif moderate_overlap (>40%):
    return 0.8
else:
    return 0.7  # Model IDs present but weak match
```

Step 3: If no model IDs in query, use fuzzy matching
```python
if brand_match and keyword_overlap > 0.6:
    return 0.7
elif brand_match and keyword_overlap > 0.4:
    return 0.5
elif brand_match and keyword_overlap > 0.2:
    return 0.3
else:
    return 0.0
```

VERSION HANDLING
----------------

Query: "KD500 2"
Extracted: ["KD500", "KD500 2"]

The version "2" is included both:
1. As standalone in the base model identifier "KD500"
2. As combined "KD500 2"

This ensures:
- Products with "KD500 2" score highest
- Products with only "KD500" (no version) fail mandatory check
- Products with "KD500 3" (wrong version) fail mandatory check

Version matching is EXACT, not approximate.

MODEL IDENTIFIER PATTERNS
-------------------------

Standard codes:
- Pattern: [A-Z]{2,}[0-9]+[A-Z]*
- Examples: KD900X, MT500, NH100, KS900, LD
- Matches: 2+ letters, followed by numbers, optional letters

Versions:
- Pattern: [model_code] [digit]
- Examples: "KD500 2", "NH100 3", "MT500 1"
- Spaces, hyphens, slashes handled flexibly

Variations handled:
- "KD500-2" → matches "KD500 2"
- "KD500/2" → matches "KD500 2"
- "KD500 2" → matches "KD500-2"
