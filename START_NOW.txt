## If You Get Errors

### Error: "Cannot connect to database"
Your PostgreSQL might not be running or credentials wrong.

Check backend/.env:
```bash
cat backend/.env | grep POSTGRES
```

Should see:
```
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=decathlon_rag
POSTGRES_USER=postgres
POSTGRES_PASSWORD=test
```

If password is wrong, update it in backend/.env


### Error: "mlx_lm not found"
Run:
```bash
cd training
source venv_mlx/bin/activate
cd ..
./start_system.sh
```


### Error: "Port already in use"
Kill existing processes:
```bash
lsof -ti:5000 | xargs kill
lsof -ti:8000 | xargs kill
./start_system.sh
```

1. You send query: "running shoes under 5000"
2. api_swagger.py receives it
3. Calls local_model_server.py on port 8000
4. Model returns: {"intent":"search","search_request":{...}}
5. api_swagger.py routes to SearchTool
