Output:

    "type": "search",
    "products": [...],
    "total": 10,
    "query": "..."
  }

### 4. ✅ Implemented Intent Routing
Flow:
  1. LocalModelClient.parse(query) → structured JSON
  2. Route based on intent:
     - intent == "search" → SearchTool.execute()
     - intent == "task" → TaskTool.execute()
     - intent == "compare" → CompareTool.execute()
     - intent == "alternatives" → AlternativesTool.execute()
  3. Return JSON result directly

## Architecture Summary

BEFORE:
  User → workflow_agent.py → Qwen3-4B → backend/api/agent.py → Tools → PostgreSQL

AFTER:
  User → api_swagger.py → local_model_server.py → Tools → PostgreSQL
         (port 5000)       (port 8000)
         
  Much simpler! One public API, one model server, clean separation.
