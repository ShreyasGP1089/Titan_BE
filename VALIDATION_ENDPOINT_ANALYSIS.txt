IDENTIFIED PROBLEMS
-------------------

1. **FULL DESCRIPTIONS SENT**
   - Decathlon descriptions: 120-250 words each
   - 25 products × 200 words = ~5000 words input
   - Estimated: 3000-5000 input tokens

2. **GENERATING REASONS**
   - "reason" field requires model to explain each decision
   - 25 products × ~20 tokens/reason = ~500 tokens
   - This is EXPENSIVE and NOT USED anywhere

3. **LARGE max_tokens**
   - 2048 tokens budget
   - Model keeps generating until done or hits limit
   - Wastes time even if output is shorter

4. **NO TIMING INSTRUMENTATION**
   - Can't measure:
     - Prompt construction time
     - Inference time
     - JSON parsing time

ESTIMATED TIME BREAKDOWN
------------------------

For 25 products:

Input tokens: ~5000 tokens
- 25 × (Name ~10 tokens + Description ~180 tokens)
- Plus system prompt ~200 tokens

Output tokens: ~650 tokens
- 25 × (decision + confidence + reason ~25 tokens each)

On Apple Silicon M1/M2 with MLX:
- Input processing: ~0.5s (fast, parallel)
- Output generation: ~30-35s (slow, sequential)
  - Qwen3-4B-4bit: ~20 tokens/sec
  - 650 tokens ÷ 20 = ~32.5 seconds

**BOTTLENECK: Output generation (95%+ of time)**

RECOMMENDED OPTIMIZATIONS
-------------------------

### 1. Remove Descriptions (Easiest, Biggest Impact)

BEFORE:
```python
desc = c.description or ""
prompt_content += f"- ID: {c.id} | Name: {c.name} | Description: {desc}\n"
```

AFTER:
```python
# Name alone is sufficient for validation
prompt_content += f"- ID: {c.id} | Name: {c.name}\n"
```

**Impact**: 
- Input tokens: 5000 → ~500 (10x reduction)
- Input processing: 0.5s → 0.1s
- Total time: Still dominated by output

### 2. Remove Reasons (Big Impact)

BEFORE:
```json
{
  "id": "...",
  "decision": "RELEVANT",
  "confidence": 0.9,
  "reason": "Product is a men's golf glove designed for..." // ← REMOVE THIS
}
```

AFTER:
```json
{
  "id": "...",
  "decision": "RELEVANT",
  "confidence": 0.9
}
```

**Impact**:
- Output tokens: 650 → 150 (4x reduction)
- Generation time: 32.5s → 7.5s (4x faster!)

### 3. Remove Confidence (If Not Used)

If confidence isn't used downstream:

```json
{
  "id": "...",
  "decision": "RELEVANT"
}
```

**Impact**:
- Output tokens: 150 → 75 (2x reduction)
- Generation time: 7.5s → 3.8s (2x faster!)

### 4. Validate Fewer Products

Your hybrid ranker is excellent. Top 10-12 products are usually sufficient.

BEFORE: Send 25 products
AFTER: Send 10 products

**Impact**:
- Output tokens: 75 → 30 (2.5x reduction)
- Generation time: 3.8s → 1.5s (2.5x faster!)
