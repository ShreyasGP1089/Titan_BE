CHANGES MADE
------------

1. ADDED TIE-BREAKER LOGIC
   Location: _find_best_combination() method
   
   Problem: When multiple combinations have the same total score, 
            the first one found was kept (often the cheapest)
   
   Solution: Added tie-breaker logic that prefers higher total cost
             when scores are equal
   
   Before:
   ```python
   if result_score > best_score:
       best_score = result_score
       best_result = result
   ```
   
   After:
   ```python
   # Update best if: higher score, OR same score but higher cost (tie-breaker)
   if result_score > best_score or (result_score == best_score and result_cost > best_cost):
       best_score = result_score
       best_cost = result_cost
       best_result = result
   ```
   
   This ensures that when products have similar quality scores, 
   we use more of the available budget rather than defaulting to
   the cheapest option.

TIE-BREAKER EXPLANATION
-----------------------

When two combinations have the same total recommendation score,
we prefer the one with higher total cost (within budget).

Rationale:
- User specified a budget they're willing to spend
- If two combinations are equally good quality-wise,
  spending more gives better value for money
- Prevents defaulting to cheapest when quality is equal

Example:
  Combination A: Helmet(₹800, score=0.85) + Gloves(₹300, score=0.75)
                 Total: ₹1100, Score: 1.60
  
  Combination B: Helmet(₹1200, score=0.88) + Gloves(₹400, score=0.72)
                 Total: ₹1600, Score: 1.60
  
  Without tie-breaker: Combination A wins (found first)
  With tie-breaker: Combination B wins (higher cost, same score)
