  - Partial results support with missing_products list

CURRENT IMPLEMENTATION DETAILS
-------------------------------

**File**: backend/tools/compare_tool.py
**Method**: _score_token_match()

**Scoring Algorithm**:

Step 1: Mandatory Model Identifier Check
  if model_identifiers exist in query:
    for each model_id:
      if model_id NOT in product name:
        return 0  // Discard immediately
  
  Brand similarity NEVER compensates for missing model IDs

Step 2: Calculate Query Coverage
  matched_tokens = query_tokens ∩ product_tokens
  coverage = len(matched_tokens) / len(query_tokens)
  
  Key insight: Extra product tokens don't hurt score

Step 3: Base Score from Coverage
  100% coverage → 90 points
   90% coverage → 80 points
   80% coverage → 70 points
   70% coverage → 60 points
   60% coverage → 50 points
   50% coverage → 40 points (minimum threshold)
   <50% coverage → 0 points

Step 4: Apply Numeric Mismatch Penalty
  Extract numbers from query and product
  Identify model ID numbers (part of mandatory identifiers)
  Identify descriptive numbers (size, age, quantity)
  
  if descriptive_numbers differ:
    score -= 10  // Soft penalty, not discard
  
  Model ID numbers are mandatory (already checked in Step 1)

Step 5: Model ID Bonus
  if model_identifiers present and ALL matched:
    score += 5  // Small boost for confirmed model match

**Threshold**: 40 points minimum

NUMERIC HANDLING
----------------

Two categories of numbers:

1. Model ID Numbers (MANDATORY)
   Examples: "900" in KD900X, "500" in MT500
   Rule: Must match exactly
   Missing → Score 0

2. Descriptive Numbers (SOFT)
   Examples: Size (5), Age (8-12), Quantity (3)
   Rule: Mismatch → -10 penalty
   Missing → No penalty

Detection:
  model_id_numbers = numbers within model identifiers
  descriptive_numbers = other numbers in text

Example:
  Query: "Kiprun KD900X Size 9"
  - "900" is in KD900X → Model ID number (mandatory)
  - "9" is size → Descriptive number (soft match)
  
  Product: "Kiprun KD900X Size 10"
  - Has KD900X? ✓ (900 matches)
  - Size differs (9 vs 10)? -10 penalty
  - Final: Base score - 10

KEY RULES
---------

1. ✅ Model identifiers are MANDATORY
   - Detect patterns: KD900X, MT500, NH100, etc.
   - All model IDs must be present
   - Missing ANY → score 0
   - No exceptions

2. ✅ Brand similarity NEVER compensates
   - "Kiprun KD900X" cannot match "Kiprun Rain Cap"
   - Even though brand matches

3. ✅ Query-relative overlap
   - Extra product tokens don't hurt
   - "Ball Training Red" has extra words but still matches "Ball"
   - Realistic for verbose catalogs

4. ✅ Numeric flexibility (descriptive only)
   - Size 5 query can match Size 4 (with penalty)
   - Better than "not found"
   - Model ID numbers still strict

5. ✅ Partial results, not exceptions
   - Return what was found
   - Report what was missing
   - Frontend shows helpful message

6. ✅ Version matching
   - "KD500 2" must match version 2
   - Don't silently substitute KD500 3
   - Return "not found" if version doesn't exist

   - Product names must be in same language as query

3. Threshold fixed at 40
   - May need tuning based on usage
