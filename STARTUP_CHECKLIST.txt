STARTUP CHECKLIST - Before Starting api_swagger.py
==================================================

## Prerequisites

Before you can start the system, you need:

### 1. ✅ PostgreSQL Database Running
   - The system needs PostgreSQL with the decathlon_rag database
   - Check: psql -h localhost -U postgres -d decathlon_rag -c "SELECT 1"
   
### 2. ✅ MLX Training Completed (for local_model_server.py)
   - The SLM (Qwen3-4B) needs to be trained with your data
   - Or at least have the base model downloaded
   
### 3. ✅ Environment Variables Configured
   - backend/.env file with database credentials
   - API keys configured


## Step-by-Step Startup Guide

### OPTION 1: Start Complete System (Recommended)
This starts BOTH the model server AND the API.

```bash
./start_system.sh
```

This will:
  1. Check MLX is installed
  2. Start local_model_server.py (port 8000)
  3. Wait for model to load (~10 seconds)
  4. Start api_swagger.py (port 5000)
  5. Run health checks
  6. Display test command

**Prerequisites:**
  - PostgreSQL running
  - MLX installed (training/venv_mlx)
  - Model trained OR base model available


### OPTION 2: Start Without SLM (Testing Only)
If you want to test the API without the model server:

```bash
cd backend
python3 api_swagger.py
```

**What will happen:**
  ✓ API starts on port 5000
  ✓ Health endpoint works
  ✓ Swagger UI works (http://localhost:5000/docs)
  ⚠️  Query endpoint will FAIL (503 Service Unavailable)
  
**Error you'll see:**
  "Cannot connect to model server at http://localhost:8000"

**This is useful for:**
  - Testing database connectivity
  - Testing Swagger documentation
  - Developing without model


### OPTION 3: Start API Only (With External Model Server)
If you have the model server running separately:

Terminal 1 - Model Server:
```bash
cd training
source venv_mlx/bin/activate
python3 local_model_server.py
```

Terminal 2 - API:
```bash
cd backend
python3 api_swagger.py
```

**Full functionality** - API will connect to model server.

Run these commands to check if you're ready:

```bash
ls -la training/outputs/qwen3_4b_lora_mlx/
```
Expected: adapter_config.json, adapters.safetensors

```bash
cat backend/.env | grep -E "POSTGRES|API_KEY|LOCAL_MODEL"
```
Expected: All variables set
If fails: Copy from .env.example and configure


## If Model NOT Trained Yet

You have 2 options:

### Option A: Train the Model (Recommended for Production)
```bash
cd training
./setup_mlx.sh              # One-time setup
source venv_mlx/bin/activate
python3 train_mlx.py --mode train --iters 300
```
Time: ~15 minutes on M1/M2/M3 Mac
Result: training/outputs/qwen3_4b_lora_mlx/

### Option B: Use Base Model (Quick Test)
The local_model_server.py will automatically fall back to the base model
if the adapter is not found. It will work but won't be fine-tuned for
your specific task.

Just start the server:
```bash
cd training
source venv_mlx/bin/activate
python3 local_model_server.py
```

It will warn: "⚠️ Base model loaded (no fine-tuning applied)"
But it will still work for basic testing.

## What Happens When You Start api_swagger.py

1. **Initialization**
   - Loads environment variables
   - Initializes LocalModelClient (connects to port 8000)
   - Sets up Flask app
   - Creates Swagger documentation

2. **On First Request**
   - Initializes PostgreSQL connection pool
   - Checks database connectivity
   - Tries to connect to model server
   - Logs health status

3. **Query Processing**
   When you send: POST /api/v1/query
   
   Flow:
   a. Validate API key
   b. Extract query from request
   c. Call LocalModelClient.parse(query)
      → HTTP POST to http://localhost:8000/parse-query
      → Model server returns: {"intent": "search", ...}
   d. Route to appropriate tool (SearchTool, TaskTool, etc.)
   e. Execute tool (queries PostgreSQL)
   f. Return results

4. **If Model Server Not Running**
   - Step (c) will fail
   - Returns 503 Service Unavailable
   - Error: "Cannot connect to model server"

## Troubleshooting

### Problem: "Cannot connect to model server"
Solution: Start the model server first
```bash
cd training
source venv_mlx/bin/activate
python3 local_model_server.py
```

### Problem: "Database connection failed"
Solution: Start PostgreSQL
```bash
# macOS with Homebrew
brew services start postgresql

# Or check if running
psql -h localhost -U postgres -l
```

### Problem: "mlx_lm not found"
Solution: Install MLX
```bash
cd training
./setup_mlx.sh
source venv_mlx/bin/activate
```

### Problem: "Model not found"
Solution: Either train model or let it use base model
```bash
# Option A: Train
cd training
python3 train_mlx.py --mode train --iters 300

# Option B: Use base model (no action needed, will auto-download)
```

### Problem: Port 5000 or 8000 already in use
Solution: Kill existing process
```bash
lsof -ti:5000 | xargs kill
lsof -ti:8000 | xargs kill
```
