CURRENT ARCHITECTURE (FINAL)
================================================================================

User
  ↓
api_swagger.py (Port 5000) - Public API
  ↓
  ├─→ LocalModelClient.parse() → local_model_server.py (Port 8000)
  │                                   ↓
  │                            Qwen3-4B Intent Parser
  │                                   ↓
  │                            Returns structured JSON
  │
  └─→ Intent Routing
      ↓
      ├─→ SearchTool
      │     ↓
      │   HybridSearch (keyword + semantic)
      │     ↓
      │   embedding.get_embedding() [LOCAL sentence-transformers]
      │     ↓
      │   PostgreSQL + pgvector
      │
      ├─→ TaskTool
      │     ↓
      │   BudgetOptimizer + SearchTool
      │     ↓
      │   PostgreSQL
      │
      ├─→ CompareTool
      │     ↓
      │   PostgreSQL
      │
      └─→ AlternativesTool
            ↓
          PostgreSQL

PERFORMANCE CHARACTERISTICS
================================================================================

Budget Optimizer:
  - Algorithm: Greedy with lookahead
  - Complexity: O(n log n)
  - Typical execution: 1-5ms
  - Memory: O(n)

Embeddings:
  - Model: sentence-transformers/all-MiniLM-L6-v2
  - Dimension: 384
  - Generation: Local (no HTTP overhead)
  - Batch support: Yes (batch_size=32)

Intent Parsing:
  - Model: Qwen3-4B + LoRA
  - Device: MPS (Apple Silicon)
  - Response time: ~200-500ms
  - Format: ChatML

Search:
  - Method: Hybrid (keyword + semantic)
  - Candidate pool: Up to 100 products
  - Final results: Top 10 (configurable)

================================================================================
STARTUP INSTRUCTIONS
================================================================================

1. Install Dependencies:
   cd backend
   pip install -r requirements.txt

   cd ../training
   pip install -r requirements_mlx.txt

2. Set Environment Variables:
   cd backend
   cp .env.example .env
   # Edit .env with your database credentials

3. Start Model Server:
   cd training
   python local_model_server.py
   # Runs on port 8000

4. Start API Server:
   cd backend
   python api_swagger.py
   # Runs on port 5000

5. Test:
   # Swagger UI
   open http://localhost:5000/docs

   # Or use curl
   curl -X POST http://localhost:5000/api/v1/query \
     -H "Api-Key: decathlon_smart_search_2024_secure_key_abc123xyz" \
     -H "Content-Type: application/json" \
     -d '{"query": "running shoes under 5000"}'

================================================================================
DEBUGGING
================================================================================

If embeddings fail:
  1. Check sentence-transformers installed:
     pip list | grep sentence-transformers

  2. Test embedding module:
     cd backend
     python -c "from embedding import get_embedding; print(len(get_embedding('test')))"
     # Should output: 384

  3. Test debug endpoint:
     curl -X POST http://localhost:5000/api/v1/debug/embedding \
       -H "Api-Key: decathlon_smart_search_2024_secure_key_abc123xyz" \
       -H "Content-Type: application/json" \
       -d '{"text": "test"}'

If model server fails:
  1. Check model server health:
     curl http://localhost:8000/health

  2. Check adapter exists:
     ls -la training/outputs/qwen3_4b_lora_mlx/

  3. Check logs in model server output

If API fails:
  1. Check health endpoint:
     curl http://localhost:5000/api/v1/system/health

  2. Check database connection in logs

  3. Verify API key in request headers
