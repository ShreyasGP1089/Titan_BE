VALIDATION ENDPOINT - ULTRA-COMPACT JSON FORMAT
================================================================================

LOCATION: training/local_model_server.py
ENDPOINT: POST /validate-products

================================================================================
WHY JSON (NOT CUSTOM FORMAT)
================================================================================

The Qwen model has been fine-tuned to output JSON.

Using a custom format (like line-based "id|decision|confidence") fights against
the model's learned output distribution, leading to:
- Inconsistent formatting (spaces, different delimiters)
- Lower reliability
- Need for increasingly complex parsers

JSON leverages the model's training and remains robust.

================================================================================
ULTRA-COMPACT JSON FORMAT
================================================================================

PREVIOUS (Verbose):
{
  "validated": [
    {"id":"8828088","decision":"RELEVANT","confidence":1.0},
    {"id":"8787845","decision":"RELATED","confidence":0.8}
  ]
}
Tokens: ~18 per product

CURRENT (Ultra-Compact):
{"8828088":["RELEVANT",1.0],"8787845":["RELATED",0.8]}
Tokens: ~6 per product

SAVINGS: 67% token reduction!

Format: {"id":[decision,confidence],...}
- Still valid JSON
- Model understands this structure
- Parser is one line: for prod_id, [decision, conf] in result.items()
- Tiny output size

================================================================================
PARSER IMPLEMENTATION
================================================================================

Simple and robust:

```python
# Parse: {"8828088":["RELEVANT",1.0],...}
result = json.loads(json_str)

validated_array = []
for prod_id, value_array in result.items():
    decision = value_array[0].strip().upper()
    confidence = value_array[1] if len(value_array) >= 2 else 1.0
    
    validated_array.append({
        "id": prod_id,
        "decision": decision,
        "confidence": confidence
    })

return {"validated": validated_array}
```

One-liner core logic!

================================================================================
OPTIMIZATIONS APPLIED
================================================================================

1. ✅ Ultra-compact JSON format
   {"id":["DECISION",conf],...}

2. ✅ max_tokens: 1024
   Sufficient headroom to prevent truncation

3. ✅ Descriptions truncated to 150 chars
   Provides context while reducing input tokens

4. ✅ Validate top 10 candidates only
   Hybrid search quality makes this safe

5. ✅ Comprehensive diagnostic logging
   - Token utilization percentage
   - Last 300 characters of response
   - Truncation warnings
   - Parse success rate

6. ✅ Confidence field kept
   Used in scoring (budget_optimizer.py, hybrid_search.py)

7. ✅ Reason field removed
   Not used downstream, wastes tokens
