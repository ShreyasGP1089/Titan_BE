================================================================================
PROBLEM
================================================================================

Error:
  AttributeError: 'NoneType' object has no attribute 'lower'

Location:
  File: backend/services/hybrid_search.py
  Function: calculate_keyword_score()

Root Cause:
  Many Decathlon products have null metadata fields:
    - category_level_1: null
    - category_level_2: null
    - description: null

  Code was calling .lower() directly on these null values:
    product.get('name', '').lower()  ← UNSAFE if get() returns None
    product.get('category_level_1', '').lower()  ← CRASHES on null

The pattern:
  field.get('key', '').lower()

Does NOT protect against null because:
  - If field contains {'key': null}, get() returns None
  - None.lower() crashes with AttributeError

Correct pattern:
  (field.get('key') or '').lower()

This handles:
  - {'key': null} → None → '' → ''
  - {'key': ''} → '' → '' → ''
  - Missing key → None → '' → ''

================================================================================
SOLUTION
================================================================================

### 1. Added _safe_lower() Helper Method

Location: backend/services/hybrid_search.py

```python
def _safe_lower(self, value) -> str:
    """
    Safely convert a value to lowercase string.
    
    Handles None, non-string types, and missing values.
    """
    if value is None:
        return ''
    if not isinstance(value, str):
        return str(value).lower() if value else ''
    return value.lower()
```

Benefits:
  ✓ Handles None values
  ✓ Handles non-string values (numbers, booleans)
  ✓ Handles missing values
  ✓ Always returns a string
  ✓ Never crashes

NULL-SAFE PATTERNS
================================================================================

### Pattern 1: Using Helper Method (Recommended)

```python
def _safe_lower(self, value) -> str:
    if value is None:
        return ''
    if not isinstance(value, str):
        return str(value).lower() if value else ''
    return value.lower()

# Usage
product_name = self._safe_lower(product.get('name'))
```

Benefits:
  ✓ Centralized null handling
  ✓ Handles non-string types
  ✓ Reusable
  ✓ Testable

### Pattern 2: Inline Handling

```python
product_name = (product.get('name') or '').lower()
```

Benefits:
  ✓ Concise
  ✓ No helper method needed
  ✓ Clear intent

When to use:
  - Simple cases
  - When helper method not available
  - One-off conversions

### Pattern 3: Defensive Programming

```python
value = product.get('field')
if not isinstance(value, str):
    value = str(value or '')
field_lower = value.lower()
```

When to use:
  - Complex type handling
  - When you need the intermediate value
  - Debugging

### Anti-Patterns (DO NOT USE)

❌ WRONG:
```python
field = product.get('key', '').lower()
```

Why: If dict contains {'key': null}, get() returns None, not default ''.

❌ WRONG:
```python
field = product['key'].lower()
```

Why: Crashes if key missing or value is None.

❌ WRONG:
```python
field = str(product.get('key')).lower()
```

Why: str(None) = 'none' (string "none"), not empty string.

✅ CORRECT:
```python
field = (product.get('key') or '').lower()
# OR
field = self._safe_lower(product.get('key'))
```
